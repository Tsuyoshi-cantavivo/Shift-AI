# インラインAI生成のドラフト保存設計

## 目的

店舗管理画面のAI生成を、通知も公開も行わない調整可能なドラフトとして保存する。AI生成後にタイムラインのバーを直接調整し、利用者が明示的に確定した場合だけシフトを公開する。

## 方針

- 「シフト管理」画面のプレビューの主ボタンは `draft: true` を送る「ドラフトとして保存して調整」にする。
- APIの既定値は既存連携を保つため変更せず、利用者画面は必ず `draft: true` を明示する。
- 確定・通知は、ドラフト保存後の既存「ドラフトを確定・通知」操作で行う。
- 既に確定済みのシフトはドラッグ対象にしない。新たにドラフト保存したAI結果だけが対象となる。

## 検証

- フロントエンドのインライン生成が `draft: true` を送信し、主ボタン文言がドラフト調整を示すことを静的テストする。
