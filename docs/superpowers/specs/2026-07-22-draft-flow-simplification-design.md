# ドラフト運用の一本化設計

## 目的

シフト作成の利用者導線を「AI生成 → ドラフト保存 → 手動調整 → ドラフト確定・通知」に統一する。確定前のAI結果を誤って公開する操作を画面からなくし、短いバーに収まらない「ドラフト」文字を表示しない。

## 変更内容

- シフト管理画面から「一括確定（auto）」のボタンとクリック処理を削除する。
- AI生成画面に残る「今すぐ確定・通知」も削除する。AI生成はドラフト保存だけにする。
- 「ドラフトを確定・通知」は唯一の公開操作として残す。
- `/api/shop/shifts/auto-confirm` と `draft:false` のAPI互換性は残す。
- ドラフトバーの破線・ドラッグハンドルは残し、CSS疑似要素の「ドラフト」ラベルだけを削除する。

## 検証

- フロントエンドの静的契約テストで、廃止するUI識別子と文言が存在せず、ドラフト確定操作が残ることを確認する。
- CSS契約テストでドラフトラベルの疑似要素が存在しないことを確認する。
- 全pytestとJavaScript構文チェックを実行する。
